# 阶段 08：通用报表模板 + 模块组合验证

## 目标
1. 预置行业通用报表模板，减少 LLM 调用，提升报表一致性
2. 验证模块组合能力（如制造+零售、贸易+轻加工）

## 具体任务

### 任务 1：通用报表模板

在 ai-gateway 中预置报表模板，每个行业最常用的 3-5 张报表：

**全行业通用**：
- 库存台账（产品/数量/金额/库位）
- 应收应付汇总（客户/金额/到期日）
- 收付款明细（日期/类型/金额/关联合同）

**制造业**：
- 生产日报（工单/计划数量/完工数量/报废数量）
- BOM 成本分析（产品/原材料/用量/单价/小计）
- 质检汇总（批次/合格率/不合格原因）

**零售业**：
- 销售日报（门店/收银员/交易数/金额/支付方式）
- 会员消费分析（会员/等级/累计消费/积分）
- 库存预警（产品/当前库存/安全库存/需补货量）

**物流业**：
- 运单汇总（状态/数量/金额）
- 车辆利用率（车辆/任务数/里程/收入）
- 仓储利用率（仓库/已用库位/总库位/利用率）

实现方式：
- 在 export-excel-tool.ts 中添加模板配置
- 用户说"导出库存台账"时，自动匹配模板列定义
- 模板按行业配置，不硬编码

### 任务 2：模块组合验证

验证以下组合是否可行（不改代码，只改配置）：

**组合 1：制造 + 零售（前店后厂）**
- 创建公司配置：同时启用 manufacturing + retail 模块
- 验证：生产完工后可以直接 POS 销售
- 检查库存是否互通（生产入库 → 零售出库）

**组合 2：贸易 + 轻加工**
- 创建公司配置：trading 行业 + 手动添加部分 manufacturing 属性
- 验证：贸易合同关联简单加工（分切、包装）
- 检查 attributes 是否灵活组合

**组合 3：项目 + 服务**
- 创建公司配置：project_based 行业 + service_provider 属性
- 验证：项目合同关联服务交付
- 检查里程碑付款 + 服务计费是否兼容

### 任务 3：写组合指南

在 INDUSTRY_AUDIT.md 追加"模块组合指南"章节：
- 哪些模块可以组合
- 组合时的配置方法
- 已知的限制和注意事项

## 成功标准

- 至少 3 个行业有预置报表模板
- 至少 2 种模块组合验证通过
- 模板可通过自然语言触发导出
- npm run check 通过
