# 阶段 05：多租户演示环境搭建

## 目标
搭建一个包含多家公司、真实数据、完整账号的可调试环境，让项目拥有者可以直接上手测试。

## 背景
这是一个多租户 ERP/CRM 系统，支持 7 个行业模板。需要模拟真实运行状态：
- 多家公司，不同行业
- 每家公司有管理员和员工账号
- 有真实的业务数据（合同、库存、付款、客户）
- 平台管理员账号
- API Key 和渠道配置就绪

## 具体任务

### 1. 创建测试公司（2-3 家）

通过后端 API 或直接操作数据库创建：

| 公司 | 行业 | 说明 |
|------|------|------|
| 鑫盛贸易 | 贸易 trading | 主力公司，数据最多 |
| 恒达制造 | 制造 manufacturing | 有 BOM/工单数据 |
| 优鲜零售 | 零售 retail | 有 POS/会员数据 |

每家公司需要：
- 在 `companies/` 目录有配置文件
- 在 `config/industry.json` 中注册
- 独立数据库文件（多租户隔离）

### 2. 创建用户账号

每家公司至少：

| 角色 | 用户名 | 密码 | 说明 |
|------|--------|------|------|
| admin | admin_{公司} | Admin@123 | 公司管理员 |
| operator | op_{公司} | Op@123 | 业务操作员 |
| warehouse | wh_{公司} | Wh@123 | 仓管员 |

平台管理员：
| 角色 | 用户名 | 密码 |
|------|--------|------|
| platform_admin | superadmin | Super@123 |

### 3. 灌入测试数据

每家公司至少：

**合同（contracts）**：5-10 条，不同状态（draft/active/completed）
**客户（customers）**：10-20 条，有联系方式
**库存（inventory）**：20-30 条，不同产品，有数量和金额
**付款（payments）**：5-10 条，关联合同
**供应商（suppliers）**：3-5 条

数据要合理：金额、数量、日期要对得上，不要出现负数库存或金额不匹配。

### 4. 配置 API 和渠道

- 后端 JWT_SECRET 确认可用
- CHAT_API_KEY 确认配置
- 前端 admin-ui 能登录
- AI Gateway 能对话

### 5. 自测验证

完成配置后，必须自测以下流程：

- [ ] 平台管理员登录 admin-ui
- [ ] 切换到不同公司
- [ ] 每家公司管理员登录
- [ ] 查看合同列表，有数据
- [ ] 查看库存列表，有数据
- [ ] 创建一个新合同
- [ ] 入库一个产品
- [ ] AI 对话能正常回复
- [ ] Excel 导入能正常工作

### 6. 写交付文档

写一个 `DEMO_GUIDE.md` 到项目根目录，包含：
- 所有账号密码
- 测试流程步骤
- 每个账号能做什么
- 预期结果

## 成功标准

- 至少 2 家公司有完整数据
- 所有账号能登录
- CRUD 操作正常
- AI 对话正常
- 有交付文档

## 时间限制

30 分钟（这个任务比一般阶段大，可以分多轮完成）
