# 阶段 06：行业能力审计与通用化梳理

## 目标
全面梳理当前系统对各行业的支持程度，明确通用能力和行业差异，输出一份可执行的行业支持策略文档。

## 背景
- 系统面向传统小公司，支持 7 个行业模板
- 设计理念：通用底座 + 模块化行业插件，低成本复制
- 当前状态：核心 6 表 + attributes JSON + 3 个行业专有模块（制造/零售/物流）+ 1 个插件（纸业贸易）
- 问题：没有人系统梳理过各行业实际需求 vs 当前能力的差距

### 3. 典型行业深度分析

**制造业（重点）**：
- BOM 多层展开：当前实现是否支持递归？成本滚加？
- 工单状态机：待排产→生产中→完工→入库，流转是否完整？
- 工序管理：在制品追踪、工序报工、工时统计，当前能做到什么程度？
- 质检：来料检/过程检/成品检，当前表结构够用吗？
- 原材料管理：采购→入库→领料→退料，流程完整吗？

**零售业（重点）**：
- POS 交易：单笔交易→日结→月结，当前支持吗？
- 会员体系：积分/等级/储值，当前表结构够吗？
- 多门店：各门店独立库存/独立收银/总部汇总？
- 促销：满减/折扣/买赠，需要什么？

**物流业（重点）**：
- 运单状态流转：待取件→已取件→运输中→派送中→已签收→异常，状态机完整吗？
- 仓储：库位管理/盘点/拣货路径？
- 车辆调度：GPS追踪/路线规划？（这部分可能用外部系统）
- 代收货款：COD 流程？

### 4. 通用化提取

分析哪些可以提升为通用能力（不需要行业定制）：

- **状态机引擎**：合同状态/工单状态/运单状态，能否用一套通用状态机配置？
- **审批流程**：合同审批/付款审批/请假审批，能否通用？
- **报表引擎**：各行业报表不同，但查询和导出逻辑能否通用？
- **通知/提醒**：到货提醒/付款到期/库存预警，能否通用？
- **文件附件**：各行业都需要，当前已通用？
- **操作日志**：变更记录，当前已通用？

### 5. 模块化边界

分析当前代码的模块化程度：
- 加一个新行业，需要改多少现有文件？
- 行业模块之间的耦合度？
- 哪些是"必须改核心代码"的，哪些是"只加配置/模块"的？

## 输出

写一份完整文档到 `INDUSTRY_AUDIT.md`，包含：
1. 当前能力总览（一张表）
2. 各行业差距分析（每行业一个表格）
3. 通用化提取建议
4. 优先级排序和路线图
5. 新公司部署指南（给不懂系统的人看的）

## 成功标准

- 7 个行业全部分析完
- 每个行业明确"能做什么、不能做什么、建议做什么"
- 通用化提取方案明确
- 有清晰的优先级路线图
- 文档让不懂系统的人能看懂如何部署新公司
